Local Storage in JavaScript

Einführung

Der localStorage sowie der sessionStorage erlauben es key / value Paare in Browser zu speichern.

Unterschied zum Cookie

Zwischen Local Storage gibt es einige Unterschiede:

  • Anders wie bei den Cookies werden die Objekte nicht bei jeder Anfrage an den Server geschickt.
  • Der Server kann zudem die Objekte nicht manipulieren.
  • Die Speicherung ist immer auf die Domain, das Protokoll sowie den Port gebunden. Die Objekte sind immer nur auf der Seite verfügbar, von welcher sie erstellt worden sind.

Wert setzen

Um einen wert zu setzen wird das Kommando localStorage.setItem('key', 'value') verwendet. Dabei stellt 'key' der Name des Speicherobjekts dar.

localStorage.setItem('test', 'hallo')
Wert lesen

Um den Wert aus dem Local Storage zu lesen wird das Komando localStorage.key verwendet. Hierbei stellt key wieder den namen des Speicherobjekts dar.

localStorage.setItem('test', 'hallo')
console.log(localStorage.test) // hallo
